# Goal Loop Task Template

## [GOAL]

Describe the concrete outcome to achieve.

## [CURRENT_COMPLETED_COMPONENTS]

List what already exists and can be reused.

## [TARGET_END_TO_END_FLOW]

Describe the complete user or system flow that should work when the goal is done.

## [WHAT_TO_IMPLEMENT]

List the specific changes, files, features, or documentation to create.

## [PRODUCT_RULES]

Capture product, design, safety, data, and workflow constraints.

## [GOAL_LOOP_WORKFLOW]

Define the iteration loop:

1. Inspect the current state.
2. Implement the next smallest useful change.
3. Run checks.
4. Record what changed.
5. Repeat until the goal is complete or a stop condition is reached.

## [STOP_CONDITIONS]

Define when to stop, pause, or ask for input.

## [CHECKS]

List the commands, manual checks, review steps, or acceptance criteria required.

## [FINAL_REPORT_FORMAT]

Define how to summarize completed work, verification, assumptions, risks, and next steps.
